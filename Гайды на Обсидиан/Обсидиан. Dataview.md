# Описание 
Dataview - плагин для работы с большими проектами, позволяющий составлять комплексные запросы к базе заметок и встраивать результаты на страницу в виде списка, таблицы или другого отображения
Один из самых важных плагинов, позволяет строить удобные таблицы, собирать задачи из разных проектов в 1 месте, вести читательский дневник

# Метаданные
Dataview оперирует метаданными заметки 

Плагин по умолчанию имеет доступ к следующим метаданным:
- file.name - название файла
- file.folder - название папки
- file.path - путь к файлу
- file.ext - расширение файла
- file.link - ссылка на файл
- file.size - размер файла
- file.ctime - время создания
- file.cday - дата создания
- file.mtime - время редактирования
- file.mday - дата редактирования
- file.tags - список тегов в заметке (вложенные теги отображаются как несколько тегов)
- file.etags - список тегов в заметке (вложенные теги отображаются как один тег)
- file.inlinks - список входящих ссылок
- file.outlinks - список исходящих ссылок
- file.aliases - список псевдонимов
- file.tasks - список задач
- file.lists - список всех элементов всех списков в заметке
- file.frontmatter - список всех метаданных из frontmatter
- file.day - дата, в случае если заметка имеет формат наименования yyyy-mm-dd, или если в заметке если пользовательское поле метаданных с название

 Так же, можно создавать собственные метаданные, Для этого нужно:
 - Использовать двойное двоеточие в паре "ключ::значение" для метаданных посреди другого текста, 
 - Или одинарное двоеточие во frontmatter "ключ::значение"
 - Например date::2024 
 Значения метаданных могут быть следующих видов
 - Текст - "привет"
 - Число: 43
 - Логические Истина или Ложь: true false
 - Продолжительность
 - Вики-ссылка [[https://ru.wikipedia.org/wiki/Объединённые_Арабские_Эмираты| ОАЭ]] 
 - Список (содержащий другие значения)
# Запросы к БД
DQL запрос состоит из следующих модулей:
- Вид отображение результатов запроса:
	- Список - **LIST** (list without id)
	- Таблица - **TABLE** (table withiot id)
	- Задача - **TASK** 
	- Календарь - **CALENDAR**
- Источник - **From** - где и как искать файлы для отображения
	-  Можно использовать следующие источники:
		- Тэги: #сейчас
		- Папки - Путь/К/Папке
		- Файл - Путь/К/Файлу
		- Ссылки: [[]] - для заметок, ссылающийся на этот файл *или* outgoing ([[]]) - для исходящих ссылок
	- Источники можно комбинировать, используя логические операторы
		- И - **AND**
		- ИЛИ - **OR**
	- Исключать с помощью НЕ ( - )
	- Объединять с помощью круглых скобок
- Фильтр - **WHERE** - условия на основе мета-данных заметок, по которым файл, найденный и отобранный из источника будет исключен из результатов
- Сортировка - **SORT** - настройка порядка отображения результатов + DESC/ASC для настройки последовательности
- Ограничение на кол-во результатов - **LIMIT**
- Группировка - **GROUP BY** - объединение результатов на основе условия
- (Для таблицы) Нарезка - **FLATTEN** - разделение результатов по отдельным папкам
Например:
```dataview
LIST FROM outgoing([[Обсидиан.]]) WHERE icontains(file.name, "П") 
```

Помимо этого, можно создавать запросы внутри строки при помощи одинарных кавычек и равно "="
