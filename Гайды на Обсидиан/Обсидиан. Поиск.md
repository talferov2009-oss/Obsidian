# Поиск в обсидиан
- Поиск в обсидиан осуществляется 2 способами
	- Быстрый переход - поиск элементов по названию, с возможностью создать заметку. Можно использовать его для добавления заметок в хранилище, предварительно убедившись что ты не создавал заметку по этой теме ранее 
	- Полнотекстовый поиск с возможностями глубокой фильтрации. Можно использовать его для поиска заметок по тегам, соответствию поисковым словам, словосочетаниям и комбинациям разных условий. Его можно открыть вручную либо с помощью горячих клавиш.
- Полнотекстовый поиск - мощный инструмент для анализа хранилища и поиска нужных связей. Он располагает инструментами настройки точных фильтров для отображения:
	- Совпадение по слову в названии или содержимом
	- Совпадение по словосочетанию ("двойные кавычки")
	- Совпадение по наименованию пути (path)
	- Совпадение по имени файла (file)
	- Совпадение по тегу пути (tag)
		- Можно добавлять и убирать теги к поисковому запросу зажимая cmd и нажимая на тег в панели тегов
	- Совпадение по содержимому (content):
		- Совпадение по словам в одной строке (line: (mix flour))
		- Совпадение по словам в одном блоке (block: (что то там))
		- Совпадение по словам в одной секции (section: (1 2))
	- Совпадение по всем todo задачам (task:)
	- Совпадение по завершенным задачам (task-done:)
	- Совпадение по незавершенным задачам (task-todo:)
- Фильтры поиска можно комбинировать друг с другом, используя логический синтаксис:
	- Логическое "и" - пробел между фильтрами Выводит те результаты, которые соответствуют всем фильтрам
	- Логическое "или" - ключевое слово OR между фильтрами. Выводит результаты, соответствующие хотя бы 1 из фильтров
	- Логическое "не" - символ тире (-) перед фильтром. выводит все результаты, которые не соответствуют данному фильтру
	- Логические операторы можно комбинировать при помощи круглых скобок ()
- Поиск имеет настройки:
	- Чувствительность к регистру
	- Сворачивание или разворачивание результатов
	- Уточнение контекста результатов поиска
	- Объяснения используемых фильтров в поисковом запросе
	- Сортировки по имени файла, дате создания и дате изменения
- Вы можете сохранить результаты поиска в виде текста или сохранить поисковый запрос в избранное для постоянного доступа к нему
- Вы можете встроить поисковый запрос в заметку. Для этого нужно создать блок кода и передать в качестве ключевого слова в его начале `query`. К этому полю применимы все поисковые правила
- Можно осуществлять поиск по открытому документу (ctrl + f) 
```query
ссылка
```
